# /sf:verify — 完成前验证

运行全面检查清单，确认可以标记完成。

## 执行步骤

1. 调用 `storeforge-verification` skill
2. 按验证清单逐项检查：
   - 代码规范：是否违反 domain 反模式、命名规范、错误处理完整性
   - 测试：L1/L2 通过率、覆盖率达标、测试数据无硬编码
   - Git：commit message 格式、变更文件完整、无敏感信息
   - 文档：API 变更已更新 OpenAPI spec、新枚举已注册
3. 全部通过 → 确认完成
4. 有失败 → 列出失败项，要求修复后重新验证

## 输出

验证报告，包含：
- 通过/失败/不适用 清单（按类别分组）
- 失败项详情与修复建议
- 连续 3 次验证失败则升级至用户

## 触发时机

- 标记任务完成前自动调用
- 用户手动调用
