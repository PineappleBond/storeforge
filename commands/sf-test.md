# /sf:test — 三层测试执行

执行 L1/L2/L3 三层测试体系。

## 执行步骤

1. 调用 `storeforge-testing` skill
2. 按 L1 → L2 → L3 顺序依次执行：
   - L1 单元测试：各端覆盖率达标（Go ≥ 80%、Vue3 ≥ 70%、Next.js ≥ 80%、Flutter ≥ 70%），race detector 必须通过
   - L2 集成测试：13 个必测场景覆盖，Docker Compose 隔离环境
   - L3 E2E 测试：Next.js 官网场景（黄金路径：浏览→搜索→详情→SSR 验证；多语言切换/SEO metadata；移动端响应式布局）
3. 每层通过后方可进入下一层，覆盖率不达标则阻断

## 输出

测试报告，包含：
- 各层通过率（通过/失败/跳过）
- 覆盖率数据（行覆盖率 + 分支覆盖率）
- 失败用例详情

## 前置条件

- 需要已有代码

## 注意事项

- Flutter 5 端平台特定测试标记为"手动 QA 必需"
- 测试数据使用 UUID 生成，禁止硬编码
